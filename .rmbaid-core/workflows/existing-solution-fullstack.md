# RMB_AID Workflow: Improve Existing Solution — Full Pipeline
# Use when: User has existing code or an approach that needs improvement

## Trigger
User says: "fix", "improve", "my code doesn't work", "results are wrong",
"optimize this", "this isn't accurate enough", "extend this solution"

## Pipeline
1. Invoke Drishti → Reframe the problem
2. Invoke Bodhi → Analyze existing solution and failure points
3. Invoke Tarka → Form root-cause and fix hypothesis
4. Invoke Yukti → Plan the improvement
5. Invoke Karma → Implement the improvement
6. Invoke Satya → Validate the improvement
7. Invoke Siddhi → Package the improved solution

## Special Instructions
- Bodhi must analyze existing code before proposing anything new
- Tarka must explain why the current approach failed
- Karma must preserve working parts and change only what is needed
